# 学习进度自动保存与恢复功能文档

## 1. 功能概述
为了提升用户体验，我们实现了**学习进度自动持久化**功能。即使用户中途强制关闭小程序或退出微信，下次进入同一章节时，程序会自动定位到上次练习的单词位置，实现无缝续学。

## 2. 技术实现

### 2.1 数据结构升级 (`utils/storage.js`)
我们在本地存储的 `progress` 对象中新增了 `chapterProgress` 字段，用于精细化记录章节内的进度。

**新的数据结构示例：**
```json
{
  "cet4": {
    "currentChapter": 1,
    "completedChapters": [0],
    "chapterProgress": {
      "0": 200, // 第0章学到了第200个词（已完成）
      "1": 45   // 第1章学到了第45个词（未完成，下次从这里开始）
    }
  }
}
```

### 2.2 核心方法
*   **updateWordProgress(dictId, chapter, wordIndex)**: 实时更新当前章节的单词索引。
*   **getChapterProgress(dictId, chapter)**: 获取指定章节上次停留的索引。
*   **markChapterCompleted(dictId, chapter)**: 章节完成时调用，记录完成状态并重置单词索引。

### 2.3 业务逻辑 (`pages/typing/typing.js`)
1.  **自动保存**：在 `loadWord(index)` 方法中，每加载一个新单词，立即调用 `updateWordProgress` 将当前 `index` 写入存储。
2.  **自动恢复**：在 `loadChapter()` 初始化章节时，优先读取本地存储的进度：
    ```javascript
    let savedIndex = storage.getChapterProgress(dictId, chapter)
    // ...
    this.loadWord(savedIndex)
    ```
3.  **循环复习**：当用户完成章节（`finishChapter`）后，我们会将该章节的进度重置为 0，确保用户下次点击该章节时能从头复习。

## 3. 本地存储性能瓶颈分析

用户询问关于本地缓存的性能瓶颈，特别是关于“多少词汇会导致问题”。由于微信小程序使用 `wx.setStorageSync` 进行同步读写，大量数据会导致 IO 阻塞，造成 UI 卡顿。

### 3.1 关键限制
*   **单条数据限制**：虽然理论上限较高，但建议单次 `setStorageSync` 写入不超过 **1MB**。
*   **总容量限制**：整个小程序的所有本地缓存总上限为 **10MB**。
*   **性能阈值**：写入超过 **200KB - 500KB** 的 JSON 字符串时，低端安卓机型可能会出现明显的掉帧（卡顿）。

### 3.2 测算模型
假设单个单词对象（包含拼写、音标、释义、统计数据）平均占用 **250 Bytes**。

#### 场景 A：错题本 (`error_words`)
错题本是一个单独的数组存储在 Storage 中。
*   **流畅范围**：< 1,000 词 (约 250KB) - 读写极快，无感知。
*   **轻微卡顿**：1,000 ~ 3,000 词 (约 250KB - 750KB) - 保存时可能偶发轻微卡顿。
*   **性能瓶颈**：**> 4,000 词** (约 1MB)
    *   **现象**：每次答错题（触发写入）时，界面可能会卡死 200ms-500ms。
    *   **风险**：接近 1MB 时可能写入失败，或者占用过多总配额。

#### 场景 B：自定义词典 (`custom_dicts`)
通常用户会创建多个词典，每个词典是数组中的一项。
*   如果一个自定义词典包含 **5,000+** 单词，加载该词典（读取）和修改该词典（添加单词）时会遇到显著性能问题。

### 3.3 结论与建议
在此架构下（全量读写 Storage），**4,000 个单词**是单个列表（如错题本或单个自定义词典）的**软性能瓶颈**。

**优化建议（未来规划）：**
如果用户数据量预计超过此量级，建议：
1.  **数据分片**：将大数组拆分为 `error_words_page_1`, `error_words_page_2` 等多个 Key 存储。
2.  **异步存储**：改用 `wx.setStorage` (异步) 替代 `wx.setStorageSync`，避免阻塞 UI 线程。
3.  **数据库方案**：使用微信小程序的云开发数据库或 SQLite 插件（如果支持），仅在内存中维护当前所需数据。
