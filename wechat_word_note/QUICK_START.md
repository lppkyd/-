# 🚀 快速预览和调试指南

## 一、立即预览（5 分钟）

### 步骤 1：下载微信开发者工具

访问：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

根据你的操作系统下载对应版本（Windows / macOS / Linux）

### 步骤 2：打开项目

1. 启动微信开发者工具
2. 选择「小程序」→「导入项目」
3. **项目目录**：选择 `qwerty-learner-mini` 文件夹
4. **AppID**：选择「测试号」（无需注册即可使用）
5. 点击「导入」

### 步骤 3：开始调试

项目自动编译后，你会看到：

**左侧**：模拟器窗口，显示小程序界面
**右侧**：代码编辑器
**下方**：调试器（Console、Network、Storage 等）

## 二、功能测试

### ✅ 测试打字功能

1. 启动后默认进入打字练习页面
2. 你会看到第一个单词（例如 "cancel"）
3. 在输入框中输入字母：
   - 正确的字母会显示绿色 ✓
   - 错误的字母会显示红色 ✗
4. 输入完整单词后自动跳到下一个
5. 点击 🔊 播放发音

**注意**：首次使用发音功能，需要勾选「不校验合法域名」：

- 右上角「详情」→「本地设置」→ 勾选「不校验合法域名、web-view...」

### ✅ 测试词典切换

1. 点击顶部「CET4 - 第 1 章 切换 >」
2. 或点击底部导航「📚 词典」
3. 可以看到 CET4 和 CET6 两个词典
4. 点击词典进入练习
5. 点击「选择章节」可跳转到指定章节

### ✅ 测试自定义词典

1. 进入「📚 词典」页面
2. 点击右上角「+ 添加」
3. 输入词典名称：`我的生词本`
4. 输入单词列表：

   ```
   apple
   n. 苹果

   banana
   n. 香蕉

   orange
   n. 橙子
   ```

5. 点击「保存词典」
6. 返回后可以看到新词典

### ✅ 测试错题本

1. 在打字练习时故意输错几个单词
2. 点击底部导航「📝 错题本」
3. 可以看到输错的单词列表
4. 点击 🔊 播放发音
5. 点击 ✕ 删除错词
6. 点击「清空」删除所有错词

## 三、调试技巧

### 查看 Console 日志

1. 点击「调试器」→「Console」标签
2. 可以看到：
   - 词典加载日志
   - 输入校验日志
   - 发音播放日志

### 查看本地存储

1. 点击「调试器」→「Storage」标签
2. 可以看到：
   - `settings` - 用户设置（当前词典、章节等）
   - `custom_dicts` - 自定义词典数据
   - `error_words` - 错题记录
   - `progress` - 学习进度

### 清除数据重新开始

1. 在 Storage 标签中
2. 选中所有存储项
3. 点击删除
4. 刷新页面

### 调试网络请求

1. 点击「调试器」→「Network」标签
2. 播放单词发音时可以看到：
   - 请求 URL：`https://dict.youdao.com/dictvoice?audio=...`
   - 状态码：200
   - 返回音频文件

## 五、常见问题

### ❌ 问题：编译报错 "Cannot read property '0' of undefined"

**原因**：词典数据加载失败

**解决**：

1. 检查 `data/cet4.json` 和 `data/cet6.json` 是否存在
2. 检查 JSON 文件格式是否正确
3. 在 Console 中运行：
   ```javascript
   const dict = require('./utils/dict.js')
   console.log(dict.getDictInfo('cet4'))
   ```

### ❌ 问题：发音播放失败

**原因**：网络请求被拦截

**解决**：

1. 勾选「不校验合法域名」
2. 或配置服务器域名：
   - 小程序后台 → 开发 → 开发管理 → 开发设置 → 服务器域名
   - 添加 `https://dict.youdao.com`

### ❌ 问题：输入框无法输入

**原因**：focus 属性问题

**解决**：

1. 点击输入框激活
2. 或修改 `typing.wxml`，设置 `focus="{{true}}"`

### ❌ 问题：页面跳转后数据丢失

**原因**：使用了 `navigate` 而非 `reLaunch`

**解决**：

- 底部导航已使用 `reLaunch`，确保数据刷新
- 如需保留数据，改用 `navigateTo`

## 六、性能优化建议

### 1. 词典加载优化

词典数据较大（~1MB），可以考虑：

- 按需加载章节数据
- 使用分包加载
- 压缩 JSON 数据

### 2. 输入性能优化

如果输入卡顿：

- 减少 `setData` 调用频率
- 使用防抖处理输入事件

### 3. 内存优化

- 及时销毁音频上下文
- 避免大量数据存储在 data 中
