# AI 协作开发实战案例：如何使用 AI 快速迭代小程序

本文档总结了本项目中用户与 AI 协作的所有对话输入，旨在为其他人提供一份“如何与 AI 共同构建/优化产品”的实战指南。

---

## 💡 开发模式总结：四步闭环法

1.  **建立规范 (Define Standards)**: 先让 AI 理解并建立项目的开发规范（AGENTS.md）。
2.  **描述意图 (Express Intent)**: 使用自然语言描述用户痛点和功能需求。
3.  **深度对齐 (Deep Alignment)**: 通过 AI 提问和用户回答，确保双方对细节的理解一致。
4.  **逐步迭代 (Iterative Building)**: 从最简单的 MVP（最小可行产品）开始，逐步增加复杂功能和视觉优化。

---

## 📥 用户原始指令流水线

### 第一步：建立开发基石
> **Prompt**: "Please analyze this codebase and create an AGENTS.md file containing: 1. Build/lint/test commands... 2. Code style guidelines... The file you create will be given to agentic coding agents (such as yourself)..."
>
> **目的**: 让 AI 彻底理解现有代码风格、项目结构和技术栈，确保后续生成的代码能完美合入。

### 第二步：提出核心痛点
> **Prompt**: "1. 用户反馈目前的 typing 界面中，下意识会点击输入框上方的单词以输入... 希望输入栏中的光标移动到最左侧... 2. 用户遇到生词时... 考虑集成外部 API 进入小程序。"
>
> **目的**: 明确业务逻辑变化和新增功能的原始需求。

### 第三步：关键决策对齐 (Q&A)
> **AI 提问后，用户回答**:
> - **关于误触**: "我希望改变输入框上方的单词格式，让用户不要误认为上方单词框是输入框，而能明确知道这只是显示用的。"
> - **关于光标**: "光标始终在最左侧（推荐）。"
> - **关于 API**: "有道词典 API（推荐）。"
> - **关于位置**: "添加单词页面（推荐）。"
>
> **目的**: 消除歧义，在多个可选方案中选定最符合用户直觉的方案。

### 第四步：确定开发节奏
> **Prompt**: "先使用免费 web api，做一个最简单的功能添加，证明这个功能有效后，再为添加付费 api 未付费 feature。"
>
> **目的**: 控制成本与风险，采用“小步快跑”的策略。

### 第五步：细化视觉与交互
> **Prompt**: "1. 单词显示区域使用渐变式，'查词'按钮手动触发，音标显示在释义输入框上方，先使用免费 web api。"
>
> **目的**: 锁定具体的 UI 布局和交互逻辑。

### 第六步：执行与知识沉淀
> **Prompt**: "执行，并在执行完后，在 doc 中保存未来付费 api 和小程序的优化方案。"
>
> **目的**: 不仅要完成代码，还要让 AI 留下“思考痕迹”，为未来的扩展打下文档基础。

### 第七步：文化与心理层面的优化
> **Prompt**: "最后再将输入框上方的英文字母显示框变为圆形，因为中国用户习惯于认为方形风格的框是输入而非仅仅显示用。"
>
> **目的**: 基于特定用户群体的心理模型进行深度 UX 优化，体现了人类在产品直觉上的核心价值。

---

## 🚀 给开发者的建议：如何更好地使用 AI

1.  **不要怕 AI 犯错，要怕描述不清**: 好的 Prompt 应该包含“背景 (Context) + 任务 (Task) + 约束 (Constraint) + 目标 (Goal)”。
2.  **利用 AI 的自我反思**: 当你犹豫不决时，让 AI 给出多个方案及其优缺点对比。
3.  **文档化**: 像本项目一样，让 AI 生成 `AGENTS.md`、`FUTURE_ENHANCEMENTS.md` 等文档。这不仅仅是记录，更是让 AI 在后续工作中“保持记忆”的关键。
4.  **关注 UX 细节**: 功能实现只是第一步，真正的产品力体现在类似“圆形 vs 方形”这种对用户心理的深度理解上。
